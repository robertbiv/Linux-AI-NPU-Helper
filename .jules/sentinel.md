## 2025-04-26 - Prevent Command Injection via Defense in Depth
**Vulnerability:** `subprocess.Popen` and `subprocess.run` calls were lacking `shell=False` in multiple areas of the codebase. Although Python defaults to `shell=False`, explicitly setting it provides defense in depth against accidental command injection during refactoring. In addition, the `_build_launch_cmd` function was using `shlex.quote` on list elements passed to `subprocess.Popen` without a shell. This passed literal single quotes to terminal emulators, causing execution failures.
**Learning:** When passing argument lists to `subprocess` without `shell=True`, it inherently handles argument boundaries. Wrapping list elements in `shlex.quote()` causes the underlying program to receive the literal quotes. Thus, `shlex.quote` is only needed when passing a full command string to `shell=True` or writing a wrapper shell script. Explicit `shell=False` is a good security hygiene pattern.
**Prevention:** Always explicitly set `shell=False` in `subprocess` calls. Only use `shlex.quote` for full strings targeting a shell environment, never for tokenized list elements targeting an execve syscall.
